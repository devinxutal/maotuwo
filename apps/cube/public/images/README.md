# 图片资源存放规范

所有的阶段形态缩略图，请将它们分门别类存放在本目录对应的子文件夹中：

- **F2L 阶段**: `public/images/f2l/`
- **OLL 阶段**: `public/images/oll/`
- **PLL 阶段**: `public/images/pll/`

## 推荐格式与命名规范
为了保证在移动端以及各种高分辨率设备下的显示清晰度，建议将形态图片处理为：
- 去除多余的边缘空白和背景的透明背景图。
- 推荐使用 **`.png`** 或 **`.svg`** 格式。 

目前的页面配置默认寻找以下命名（以数字顺序排布为主，如果是 SVG 请自行在代码中批量将 `.png` 替换查阅后缀即可）：
- **F2L**: `1.png`, `2.png` ... `41.png`
- **OLL**: `1.png`, `2.png` ... `57.png`
- **PLL**: `1.png`, `2.png` ... `21.png`（或者使用习惯字母表示如 `Aa.png`, `Ub.png`，只需去对应 `src/data/*.ts` 数据结构中更改 `imgSrc` 的指向即可）。

目前的 React 代码逻辑中，默认会尝试抓取例如 `/images/f2l/1.png`。如果图片不存在，暂时会由灰色占位框替代。
