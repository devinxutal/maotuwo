# CP-3.0: Formula-specific Image Spin Support

## 1. 核心目标 (Purpose)
使得应用深度支持「相同魔方标态，不同的持握切入角度」的硬核玩法。支持针对每个候选公式配置特定旋转角度 (spin) 属性。

## 2. 需求覆盖范围 (Requirements)
### 2.1 数据与规格向后兼容
- **结构扩展**：修改 `Formula` 接口定义，兼容 `spin?: number` 可选属性。如果老的 JSON 配置文件读取解析时缺失此属性，默认视作 `0`，完美实现向旧版备份向下兼容。

### 2.2 多栖全域 UI 挂载联动
- **公式图库索引 (CaseList Grid)**：
  - 在诸如 F2L / OLL / PLL 等全局列表页，缩略图不仅展示标准的占位状态，还要判断系统目前该状态激活的 **Main Formula (主公式)**。提取主公式拥有的 `spin` 属性，让预览图默认处于玩家最习惯的角度。
- **编辑态交互控件 (Edit Mode)**：
  - 在详情页 `isEditing` 开启时，主图谱右侧新增一个直观好用的旋转控制按钮（Icon 为圆环旋转箭头，中心辅以数字文案如 90°）。
  - 点击按钮以 90 度为步长顺时针切换：`0 -> 90 -> 180 -> 270 -> 0`。触发保存后写入 Context。
- **公式详情视图 (CaseDetail Main Display)**：
  - 核心标态大图必须完全受控于当前**展示或被选中**的公式配置（不单单局限于主公式），实时旋转对应的度数。
- **公式备选罗列区 (Alternative Formulas List)**：
  - 在下列表单清单中，除公式纯文本外，各个单元格项的最左侧额外前置一个小型预览图（Mini Thumbnail），并基于其自身的 `spin` 赋予独立展示级旋转角度，直白对比。

## 3. 验收标准
- [ ] 备份格式 `JSON Spec` 文档成功支持及备案了 `spin`。
- [ ] 新老 JSON 和已有 LocalStorage 均可无缝读取不爆红。
- [ ] 所有大列表图鉴都听指挥根据主公式发生了偏移。
- [ ] 详情列表每一条公式前的微缩图完美展示其独立的触发角度。
