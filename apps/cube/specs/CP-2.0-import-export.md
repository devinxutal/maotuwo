# CP-2.0: App Data Import & Export (JSON)

## 1. 核心目标 (Purpose)
鉴于数据依靠浏览器 `LocalStorage` 保存，当用户希望换浏览器或者防备缓存丢失时急需备份手段。因此建立一整套严谨易用的全局配置 JSON 导出与恢复导入功能系统。

## 2. 功能要求 (Features)

### 2.1 导出机制 (Export)
- 提取所有的持久化配置状态。
- **数据修剪（极简结构）**：不需要导出 `imgSrc`, `name`, `alias` 这种完全能够在前端硬代码中恢复的基础数据源。仅提取每种 Case 被用户编辑保存下来的精华 —— 即 `formulas` 数组（表达式与标签）和当前激活的 `mainFormulaId`。
- **格式前瞻性兼容**：JSON 不能是个光秃秃的 Array，而应在最外围定义 `{ "version": 1, "data": {} }`。方便以后接入诸如 "黑暗模式配置" 甚至 "用户信息" 这种顶级设置。
- **生成下载**：将其格式化为具有严格锁进缩进的美化 JSON（Human Readable），并结合浏览器下载沙盒抛出下载文件命名为 `cube_backup_YYYY-MM-DD.json`。

### 2.2 导入机制 (Import)
- 构建一个文件选取框或拖拽区域，读取用户选取的 JSON 文本内容。
- **结构化合并校验**：检查输入文件的 `version` 和结构规范。解析之后，不能暴力的抛弃本地硬编码数据结构去全盘用旧数据替代；
- 正确的“无损导入”：系统读取本地当前的结构大底座（即那数十个 `CubeCase` 骨架），如果在被导入的 `data` 表中发现了完全相同的 `id`，仅仅用外部的 `formulas` 和 `mainFormulaId` 对本地系统项实行覆盖映射。
- 解析完成后，全局调用 `setCases` 并通过依赖自动落库。

## 3. UI 交互推荐 
- 最好放置于 `App.tsx` 页面最顶层全局 Header 内的角落（或者右上角的极简图标），点击后通过原生的 Modal 弹出或者滑动弹窗，提供十分清晰的操作指导界面和上传按键。

## 4. 验收标准
- [ ] 点击「导出」后能在本机文件夹自动生成一个结构良好、干净、包含所有主副公式标签配置的实体 JSON 包。
- [ ] 备份格式排除掉了所有静态图文地址等冗余信息。
- [ ] 新开一个没有任何记录配置的浏览器，点击「导入」，一旦选取以前的 JSON 包，系统图鉴数据全部被瞬间恢复激活！
