# CP-1.1: F2L Phase Forms Display

## 1. Purpose
完善 F2L 阶段的所有形态展示。用户从首页点击进入 F2L 后，能够以友好的移动端视觉布局（如两列 Grid 网格）清晰地浏览该阶段各种标准形态的列表。

## 2. 交互与布局要求
- **自适应网格**：形态卡片应当以网格排列，并且根据设备/屏幕宽度的不同自动响应，确保一行能展示 **3-5** 个形态卡片。
- **卡片构成**：每一项包含一张形态示意图（现阶段由于无真实素材可先放占位图）以及对应的名称或编号。
- **行为**：此时点击形态卡片只会提供一个明显的点按缩放反馈，暂时不跳转页面（具体对应公式将在CP-1.3实现）。

## 3. 验收标准
- [ ] 进入 F2L 页面能够看到长列可滑动的各种形态网格。
- [ ] 不同屏幕宽度下网格依然表现合适，一屏能放下合理的数量（每行 3 至 5 个）。
- [ ] 顶部的后退交互和标题栏仍然表现一致。

## 5. Dependencies
- 暂不新增外部依赖。可配合 TailwindCSS 的 `grid` 类快速构建布局。

## 6. Implementation Notes
- 构建了 `F2LPage` 组件，使用 Tailwind 的 `grid-cols-3 sm:grid-cols-4 md:grid-cols-5` 配置达到移动端 3 列，稍宽屏幕 4-5 列的动态自适应。
- 配置了由纯色及“Img”文字组成的自动图片降级（Fallback）逻辑。
- 创建了 `public/images/README.md`，用以指导用户摆放真正的 `.png` 资源图片。
